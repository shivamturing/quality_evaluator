## Evaluation Criteria

## Scoring
- Use a numeric score from **0 to 5** in **0.5** increments.
- **5** means the criterion fully meets all pass conditions.
- **0** means the criterion completely fails.
- Use intermediate scores for partial compliance.
- Set `overall_score` to the **minimum** of the four criterion scores.
- Set `overall_pass` to **true only if** `overall_score` is **5**.

### 1. Tool Selection
- **5**: Tool matches user intent AND is the most efficient/direct tool available.
- **0**: Using a generic tool when a specific one exists (e.g., `update_user` instead of `ban_user` if `ban_user` exists).
- **0**: Using multiple tools to achieve what one tool could do.
- **0**: Using wrong tool for the task (e.g., using get_user_info when search_users is needed)

### 2. Parameter Accuracy
- **5**: Parameters are 100% accurate. No fabricated values.
- **0**: "Magic inputs" (e.g., using a User ID that wasn't retrieved in a previous step)
